# Constructors in OOP (C++) — Complete Deep Guide

Constructors are one of the most important concepts in Object-Oriented Programming (OOP).

They are fundamental for:

* object initialization
* memory safety
* resource management
* large-scale software engineering

Without constructors,
objects may start in:

# invalid or garbage states

---

# SIMPLE DEFINITION

A constructor is:

# a special member function automatically called when an object is created

---

# BIG IDEA

When object is created:

```cpp id="m4v8tb"
Car c1;
```

the object needs:

* initial values
* setup
* memory/resource initialization

Constructors perform this automatically.

---

# REAL-WORLD ANALOGY

Imagine buying new phone.

Before use:

* operating system installed
* settings initialized
* memory configured

Constructor performs similar initialization for objects.

---

# WHY CONSTRUCTORS EXIST

Without constructors:

```cpp id="x7m2wc"
class Student {

public:
    int marks;
};
```

Creating object:

```cpp id="u1x5tb"
Student s;
```

may produce:

```text id="q9m3tb"
garbage value in marks
```

Dangerous.

---

# CONSTRUCTOR SOLUTION

```cpp id="r5v8qc"
class Student {

public:

    int marks;

    Student() {
        marks = 0;
    }
};
```

Now objects start safely.

---

# IMPORTANT FEATURES

Constructors:

* have same name as class
* have no return type
* run automatically
* initialize objects

---

# BASIC CONSTRUCTOR SYNTAX

```cpp id="t2m7wc"
class ClassName {

public:

    ClassName() {

    }
};
```

---

# FIRST COMPLETE EXAMPLE

```cpp id="v8x1tb"
#include <iostream>

}
```

```text id="w4m9qc"

```

# `this` POINTER

Very important.

Inside class:



# COPY CONSTRUCTOR

Very important advanced beginner topic.

---

# PURPOSE

Creates object from another object.

---

# EXAMPLE

```cpp id="m1v8tb"
class Test {

public:

    int x;

    Test(int val) {

        x = val;
    }

    Test(const Test &obj) {

        x = obj.x;
    }
};
```

---

# OBJECT COPYING

```cpp id="r6m3wc"
Test t1(10);

Test t2 = t1;
```

Copy constructor called.

---

# WHY COPY CONSTRUCTORS IMPORTANT

Critical for:

* object copying
* memory safety
* resource management

---

# SHALLOW COPY vs DEEP COPY

Extremely important concept.

---

# SHALLOW COPY

Copies:

```text id="u2x9tb"
memory addresses only
```

Dangerous for dynamic memory.

---

# DEEP COPY

Creates:

```text id="x5m1qc"
independent memory copy
```

Safer.

---

# DYNAMIC MEMORY PROBLEM

```cpp id="y8m4wc"
int *ptr = new int(10);
```

Simple copying may cause:

* double delete
* dangling pointers
* crashes

---

# RULE OF THREE

If class manages resources:
usually implement:

* destructor
* copy constructor
* copy assignment operator

Advanced but very important.

---

# DESTRUCTORS

Opposite of constructors.

Called automatically when object destroyed.

---

# SYNTAX

```cpp id="q1v7tb"
~ClassName() {

}
```

---

# EXAMPLE

```cpp id="u4m2qc"
class Test {

public:

    ~Test() {

        cout << "Destroyed";
    }
};
```

---

# WHY DESTRUCTORS IMPORTANT?

Cleanup:

* memory
* files
* sockets
* GPU resources

---

# CONSTRUCTOR CALL ORDER

In inheritance:

```text id="v9x5tb"
Parent constructor first
Child constructor second
```

---

# EXAMPLE

```cpp id="m2v8qc"
class Animal {

public:
    Animal() {
        cout << "Animal";
    }
};

class Dog : public Animal {

public:
    Dog() {
        cout << "Dog";
    }
};
```

Output:

```text id="r7m1wc"
AnimalDog
```

---

# WHY?

Base class must initialize before derived class.
